# 記憶 × 英文練習產品架構（草案）

> 依 MemoBrain《記憶天才量表訓練方案》的 8 種天份設計。**先以記憶 + 英文為主。**
> 本文是產品藍圖；對應的「測驗後 UI」已實作在
> `quizzes/memory-genius-quiz/`（結果頁的「🧠 你的記憶訓練方案」區塊）。

---

## 1. 一句話定位

不是又一個背單字 App，而是：**先測出你的「記憶天才類型」，再據此決定你「怎麼編碼、怎麼提取、怎麼複習」英文**。同一批單字，8 種類型有 8 種最有效的記法。

## 2. 記憶科學核心（來自文檔）

每個記憶項目都走三段流程，**每種類型的最佳做法不同**：

| 階段 | 意義 | 產品要做的事 |
|---|---|---|
| **編碼 Encode** | 資訊如何進入記憶 | 依型態給「編碼模板」（場景／框架／聲音／故事／類比／原理／開口／圖像）|
| **提取 Retrieve** | 主動回想 > 重看 | 依型態選提取形式（說出來／畫出來／解釋／跟讀），不是翻筆記 |
| **鞏固 Consolidate** | 間隔複習對抗遺忘 | **每型態專屬的間隔節奏**（見下表），每次複習都用「主動提取」 |

**各型態間隔複習節奏（天）：**

| 類型 | 波段 · VARK | 節奏 | 提取主形式 |
|---|---|---|---|
| 探索者 EXPLORER | θ · K | 1 → 3 → 8 → 20（密集，衰減快）| 情境演出 |
| 建築師 ARCHITECT | β · R | 1 → 7 → 21 → 60（標準，衰減慢）| 合書輸出 |
| 旋律人 MELODIST | α · A | 1 → 4 → 10 → 28 | 跟讀複述 |
| 敘事者 NARRATOR | α+θ · A+K | 1 → 5 → 14 → 35 | 口說故事 |
| 織網者 CONNECTOR | α · A+R | 1 → 5 → 15 → 40 | 教學／連結 |
| 分析師 ANALYST | β · R+K | 1 → 7 → 21 → 60 | 費曼解釋 |
| 表演者 PERFORMER | γ · K+A | 1 → 3 → 9 → 25 | 即興錄音 |
| 圖像家 VISIONARY | γ · V | 1 → 6 → 18 → 45 | 圖像重建 |

## 5. 型態自適應引擎（產品的核心差異化）

一張映射表驅動所有模組：

| 類型 | 編碼模板 | 練習模式（每日台）| 提取形式（複習）|
|---|---|---|---|
| 探索者 | 詞彙 → 場景 | AI 情境角色扮演 | 關材料、演一遍場景 |
| 建築師 | 詞彙 → 樹狀框架 | 語法地圖 + 合書輸出 | 從節點生出例句 |
| 旋律人 | 詞彙 → 音標/節奏 | Shadowing 跟讀 | 遮稿複述 |
| 敘事者 | 詞彙 → 人物故事 | 口說故事日記 | 對 AI 說故事 |
| 織網者 | 詞彙 → 類比/連結 | 概念地圖 + 費曼 | 教 AI、延伸節點 |
| 分析師 | 詞彙 → 原理/為什麼 | 費曼解釋日 | 不看材料解釋規則 |
| 表演者 | 詞彙 → 開口說三遍 | 即興對話 + 錄音 | 回聽錄音找盲點 |
| 圖像家 | 詞彙 → 圖像/記憶宮殿 | 視覺卡 + 空間放置 | 重建畫面再說詞 |

> 註：AI 教學風格已在 `supabase/functions/chat/index.ts` 依 8 型態實作（PR #7，限英文）。

## 6. 間隔複習（SRS）引擎 — MVP 規格

- 每個 `MemoryItem`（詞／句／語法點）有 `intervalIndex` 與 `nextReviewAt`。
- 初始節奏 = 該用戶型態的 `schedule`（如探索者 `[1,3,8,20]`）。
- 到期項目進入當日佇列；複習以**該型態的提取形式**呈現（說／畫／解釋／跟讀）。
- 使用者自評或 AI 判定成功 → `intervalIndex++`（前進）；失敗 → 回退一格、重排近日。
- 走完節奏 → 標記「已鞏固」，低頻抽查。
- **加值**：接 Brain Lab「最佳時段熱圖」→ 把當日佇列推到用戶專注時段。

## 7. 資料模型（先 localStorage，後接 Supabase）

```ts
GeniusProfile { primary, secondary, calibratedAt }            // 已有: memo_genius_result
MemoryItem {
  id, english, meaning, note,
  encodeMode,           // 依型態預設：scene|frame|sound|story|link|principle|speak|image
  encodePayload,        // 例：探索者的場景、圖像家的畫面
  intervalIndex, nextReviewAt, status,  // SRS 狀態
  createdAt
}
ReviewLog { itemId, date, retrieveMode, success, latencyMs }  // 算保留率/提取成功率
SessionLog { date, minutes, mode, itemsTouched }              // 練習量
```

**對接現有資產**：`memo_genius_result`（型態）、VARK profile、Brain Lab 訓練紀錄／腦態、Practice AI 適配。

## 8. 分階段落地（記憶 + 英文優先）

- **Phase 1 · MVP**：測定 → 訓練方案 → 依型態每日練習 → 基本 SRS 佇列（localStorage，純英文）。目標：跑通「測 → 練 → 複習」閉環。
- **Phase 2**：記憶卡庫 + 型態化編碼模板 + 多種提取形式（說／畫／解釋）+ 數據儀表板（保留率）。
- **Phase 3**：Brain Lab 最佳時段排程、BINA 音訊前置（旋律人睡前 δ 鞏固）、學習報告／認證。

## 9. 成功指標

提取成功率 · 7/30 天保留率 · 複習依從率（到期完成 %）· 每日練習分鐘數 · 次日留存。
